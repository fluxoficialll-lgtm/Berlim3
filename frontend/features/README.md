# Features

Este diretório contém componentes, hooks e lógica que são específicos de uma determinada funcionalidade da aplicação. O objetivo é agrupar todo o código relacionado a uma feature em um único local, tornando-o mais coeso e fácil de encontrar.

## Estrutura

Cada subdiretório dentro de `features` representa uma funcionalidade específica. Por exemplo:

- **`auth`**: Contém componentes relacionados à autenticação, como formulários de login e registro.
- **`groups`**: Contém componentes e lógica para gerenciar grupos, como listas de membros, configurações de grupo, etc.
- **`reels`**: Contém a funcionalidade de vídeos curtos, incluindo o player de vídeo, comentários e interações.

Essa abordagem ajuda a escalar a aplicação, mantendo o código organizado e modular.
